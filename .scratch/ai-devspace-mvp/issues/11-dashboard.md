# 11 - 首页 Dashboard

## 目标

落地 PRD §5.1。一眼看到所有需求状态 + 当前活跃会话 + 待办。

## 范围

- [ ] Web 端 `/`（Dashboard）三块：
  - **进行中需求卡片网格**：按 status 排序，状态色卡 + 进度环
  - **当前活跃会话**：实时显示"AI 正在做什么"（来自 WS），含「查看对话 / 打开 IDEA / 查看 Diff」快捷按钮
  - **待办事项**：AI 提问待回答、需用户决策的项
- [ ] 顶部统计：进行中 X 个 / 等待中 Y 个 / 本周完成 Z 个
- [ ] 卡片点击直达对应需求详情页
- [ ] 实时刷新：WS 推送需求状态变化时自动更新卡片
- [ ] 空状态引导（首次使用时显示"创建你的第一个需求"）

## 验收

- 3 个不同状态的需求能正确分组展示
- 活跃会话能实时反映 AI 当前任务
- 待办列表能正确显示"AI 提问待回答"等条目
