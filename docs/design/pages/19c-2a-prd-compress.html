变体 A：上下分栏 (PRD 压缩 + 辅助文件占下半) —— 点击辅助文件卡片后，PRD 顶置区压缩为标题条(~80px，仅标题 + 锚点条 + 保存状态)，下半部分区变成该文件的查看/编辑区。顶部面包屑改成 "需求 / 退款功能优化 / 📐 api-draft.md"，右上 "← 返回 PRD"。

交互流：
  1. 默认进入 DRAFTING：PRD 顶置(1.5x) + 辅助材料卡片条(1x)
  2. 点击 api-draft.md 卡片 → PRD 压缩条 slideDown 0.3s + 下半编辑器 fadeIn 0.25s
  3. 用户编辑 api-draft.md，独立保存
  4. 点 "↗ 展开 PRD" 或 "← 返回 PRD" → 上下分栏比例恢复
优点：与 PRD 顶置型整体一致；压缩态 PRD 一眼能看见标题和大纲；用户随时知道 PRD 还存在。
代价：压缩条信息密度高（标题 + 锚点 + 按钮挤在 80px）；切换时 PRD 不可见，编辑 PRD 需要展开动作。
