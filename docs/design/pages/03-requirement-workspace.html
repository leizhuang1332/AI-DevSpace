01-database 缺索引

由 design-stage 生成 · 2026-07-08 · 14 次修订 · ✓ 已采纳

1. 退款表 refund_order

主表，记录每笔退款订单的状态、金额、退款渠道等信息。一笔原订单（order_main）可以对应多笔退款订单（部分退款场景）。

字段说明

字段类型说明
idBIGINT PK主键，雪花算法
order_idBIGINT原订单 ID
user_idBIGINT用户 ID
amountDECIMAL(10,2)退款金额（元）
statusTINYINT1-待审核 2-退款中 3-成功 4-失败
reasonVARCHAR(256)退款原因
created_atDATETIME创建时间
updated_atDATETIME更新时间
💡 AI 建议：高频查询场景 WHERE user_id=? AND status=? ORDER BY created_at DESC，建议加联合索引 idx_user_status_created (user_id, status, created_at)。需优化

2. 退款流水表 refund_flow

记录退款链路上的每一步状态变更（异步、回调、重试）。

3. 索引设计

  • idx_user_status_created (user_id, status, created_at) — 用户维度查询
  • idx_order (order_id) — 原订单维度
  • idx_status_updated (status, updated_at) — 后台扫描任务
AI 提示
✨ 缺索引建议
检测到 WHERE user_id + status 查询无合适索引，建议在 refund_order 表加 idx_user_status_created
