退款功能优化

背景

用户发起退款申请后，当前需人工审核，平均耗时约 2 天，投诉率高。

目标

  • 退款流程自动化，减少人工介入
  • 提升退款到账速度与成功率

验收标准

  • 退款成功率 ≥ 99%
  • 平均退款时长 ≤ 30s
  • 失败自动重试并通知用户

非目标

  • 不改动原有订单状态机
  • 不涉及跨境退款流程

方案 ③ IDE 文件树型 —— 借鉴 VSCode 的"左侧文件树 + 右侧编辑器"模型，PRD 只是被默认选中的文件。
  1. 左 260px：完整文件树（含 PRD + 辅助文件 + 仓库 + Skill 一体化导航）
  2. 中 1fr：编辑器（顶 Tab 切换已打开文件 + 主区 Markdown 编辑 + 实时预览分屏 + 底部 status bar）
  3. 右 140px：候命 Skill 紧凑卡片
  4. 仓库勾选：沉到文件树底部"关联仓库"区，状态点直接显示
  5. 入口：launch 按钮沉到编辑器底部 status bar 右侧
优点：与开发者日常 IDE 心智完全一致；多文件切换丝滑；编辑+预览分屏；可扩展性强（后续接 Monaco / Y.js 协作）。
代价：信息密度极高，对非技术用户不友好；需要为编辑器 Tab 设计一套状态管理；首次进入时"默认打开 prd.md"的引导要做。
