决策提示:
· 如果真实场景中多会话是为了 "对比差异" → B 或 C
· 如果是为了 "分阶段深化"(先架构角度,再数据角度) → A 或 C
· 如果是为了 "保留历史方便回看" → D
· 如果你希望 "复杂度按用户需求伸缩" → C (我的推荐)

关键约束: ANALYZING 是 6 工位之一,主区不能过度复杂。A 最稳,C 最灵活,B 最激进,D 与现有 ZoneBar 视觉冲突。

⌥A ⌥B ⌥C ⌥D 在 UI 上也可作为快捷选项按钮。
