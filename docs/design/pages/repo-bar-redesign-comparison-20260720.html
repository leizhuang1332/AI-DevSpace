× 取消关联按钮的三种承载方式:
  • A: hover avatar 时右上角浮出红色 × 圆点(类 GitHub reviewer 头像)。优点:无视觉噪音;缺点:必须 hover 才能点,移动端不可达(目前桌面优先,可接受)。
  • B: 展开后 chip 右侧常驻 ×(对齐现有 chip 形态,改动最小)。优点:可视化、可发现;缺点:占展开空间,但展开本身已经是按需动作,可接受。
  • E: 气泡内 chip 右侧 ×(只在打开 FAB 时可见)。优点:展开态空间充裕;缺点:FAB 折叠时无法操作。
三个方案各自的代价:
  • A · 瘦身: 需要 chip → avatar 的视觉迁移,改动 .repo-bar + 新增 avatar 组件 + tooltip。repo name 长度信息丢失(只能 tooltip)。
  • B · 折叠: 改动 .repo-bar 一个文件 + 新增折叠面板状态。视觉延续性最强,与 issue 08 视觉基线兼容。
  • E · FAB: 推翻 issue 08 验收 #3 "sticky bottom · 工作区滚动时跟随可见"。Launch 按钮也需要重新找位置(气泡内 or 顶部 toolbar)。改动最大,但视野代价最小。
推荐方案 B(折叠 sticky)。理由: (1) 视野代价稳定 40px,与当前实现相比 -38%~-70%,且不再随 N 增长; (2) 保留 issue 08 验收 #3 的 sticky + Launch 始终可见承诺; (3) × 取消关联按钮承载在展开态 chip 上,与现有视觉基线对齐; (4) 改动范围最小(只动 .repo-bar 一个文件)。 若希望视野代价进一步降到 0,可考虑 E(FAB),但需承担 Launch 按钮重定位和 issue 08 验收 #3 调整的代价。
