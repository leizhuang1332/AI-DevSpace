方案 ② PRD 顶置型 —— 沿用当前 DRAFT 的表单风格，但
  1. 去掉 AC checklist 与 launch 按钮（AC 下沉到 ANALYZING 阶段的"准入校验"），保留标题 + PRD Markdown
  2. 下半部分追加"辅助材料"卡片网格，上传/新建文件
  3. 资源树上半显示 PRD 章节大纲（点击定位），下半显示辅助文件导航
  4. 底部固定仓库勾选条 + "进入 ANALYZING"按钮
优点：保留表单驱动感，PRD 始终可见 + 可编辑；用户心智负担最低；与现状改动最小。
代价：PRD 与辅助文件视觉权重仍不平等（PRD 占据上半 ~60%）；自动保存逻辑同时管理两类内容时需小心。
