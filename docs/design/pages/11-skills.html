analyze-stage
v1.0 · 内置
BUILT-IN
读取 requirement.md，生成需求理解（01-understanding.md）和澄清问题（02-questions.md）。理解不清的地方必须提问，不能假设。
📥 注入：requirement.md 📤 产物：analysis/*
design-stage
v1.0 · 内置
BUILT-IN
基于分析结果生成数据库、API、服务层三层设计。必须产出 OpenAPI yaml 和 SQL DDL。
📥 注入：analysis/ + knowledge 📤 产物：design/* + artifacts/*
plan-stage
v1.0 · 内置
BUILT-IN
将设计分解为可执行的 Task 列表（tasks.md）。每个 Task 包含验收标准、关联产物、预估工作量。
📥 注入：design/* 📤 产物：plan/tasks.md
code-stage
v1.0 · 内置
BUILT-IN
逐个执行 Task，写入代码到对应 repo 的 worktree。完成后自动 commit（需用户授权）。遇到阻塞自动生成问题。
📥 注入：plan/ + design/ + 当前 Task 📤 产物：commit + 更新 tasks.md
company-review-stage
v0.3 · 用户自定义 · 1 个月前
USER
遵循公司 review 规范的代码审查 Skill。检查命名规范、异常处理、日志格式（基于 knowledge/standards/java-code-spec.md）。
📥 注入：code diff + standards 📤 产物：review/*.md
db-migration-stage
v0.1 · 用户自定义 · 2 周前
USER
数据库迁移脚本生成。读取 SQL DDL，自动生成 Flyway / Liquibase 兼容的版本化迁移脚本（含回滚段）。
📥 注入：artifacts/*.sql 📤 产物：repos/*/db/migration/V*.sql
设计说明：Skill = 「AI 在某阶段如何工作」的可加载单元。MVP 内置 6 个（对应 Vibecoding 7 步流程的核心），用户可自定义扩展。每个 Skill = 一个目录，含 SKILL.md（提示词模板）+ context.yml（上下文装配规则）+ artifacts.yml（期望产物清单）。点击 Skill 卡片 → 打开完整编辑器（YAML + Markdown 双 tab）。空态：「🤖 加载 Skill · 6 个内置 Skill 已就绪」。
