  1. 7 Tab 排序:OVERVIEW → DRAFTING → ANALYZING → CLARIFYING → DESIGNING → EXECUTING → WRAP-UP(Overview 排第一)
  2. OVERVIEW 不是工位:用 brand 紫色区分于工位状态色,仪表板不是工作台
  3. 工位 = 独立路由(选项 A):6 个工位各自对应独立 URL,工位切换 = 浏览器路由跳转
  4. 排序 = lifecycle 静态顺序(子决定 5a):DRAFTING → ANALYZING → CLARIFYING → DESIGNING → EXECUTING → WRAP-UP,稳定可预测
  5. 视觉反馈 = 状态色点 + 激活态下划线(子决定 5b):无数字徽章(对齐决策 22),紫色底部 2px 下划线标识当前工位
  6. Cmd+K 双通道:Tab 给视觉反馈,Cmd+K 给键盘流快速切工位,0 增量 UI
  7. Inline 栏下放(选项 C):仅 DRAFTING / EXECUTING 保留,本原型 EXECUTING 工位右侧展示 Inline 栏
  8. 资源树按工位变化:EXECUTING 工位资源树 = 任务 DAG + 变更文件 + 产物;DRAFTING 工位资源树 = PRD + AC + 关联仓库
