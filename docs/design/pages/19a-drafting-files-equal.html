方案 ① 文件平等型 —— DRAFTING 主区是文件卡片网格，PRD 与所有辅助文件平级展示，没有「主表单区」的概念。PRD 唯一特殊身份是 badge: PRD + border: brand。

核心交互：
  • 点击卡片 → 打开文件查看/编辑（可走模态或侧滑）
  • 右上 ⋯ → 重命名 / 删除 / 导出 / 查看历史
  • 虚线卡片 → 唤起「新建/上传」菜单（md / docx / pdf）
  • 仓库勾选条固定在底部，避免遮挡文件
  • 资源树退化为「文件分组 + 仓库 + Skill」导航，不再显示 PRD 大纲
优点：所有文件同等重要，避免 PRD 抢焦；新用户上手无门槛。
代价：PRD 编辑入口要点击 1 次卡片才能进入；表单驱动感变弱。
